# Lights Out — Design

## Game concept

Lights Out is the classic electronic grid puzzle (Tiger Electronics, 1995). The
board is a 5×5 grid of lights, some lit. Pressing a light toggles it **and** its
four orthogonal neighbours. The goal is to switch **every** light off in as few
presses as possible. Clear the board to advance to the next, harder level.

Everything is drawn with the HTML5 Canvas 2D API: rounded tiles that glow when
lit and sit dark and recessed when off.

## Mechanics

- **Toggle rule** — pressing cell `(r, c)` flips that cell plus its up, down,
  left and right neighbours (neighbours off the board are simply ignored, so
  edge and corner presses flip fewer cells).
- **Solvability** — every puzzle is generated by starting from an all-off board
  and applying a number of *random presses*. Because a press is its own inverse
  (pressing the same cell twice returns the board to its previous state), any
  board reachable from all-off is guaranteed solvable by replaying those presses
  — the player can never be handed an impossible board.
- **Difficulty** — the number of scrambling presses grows with the level, so
  later boards start more tangled.
- **Scoring** — the move counter tracks presses for the current board. Your
  **best** (fewest moves to clear any board) is saved to `localStorage` under
  `lightsout-best`.
- **Winning** — when the last light goes out, the level is solved; advance to a
  fresh, harder board.

## Deterministic core (for testing)

State is a plain 5×5 boolean array `grid[r][c]` on the global scope, alongside
`moves`, `level`, `best` and `state`. The rules live in small pure functions:

- `toggle(r, c)` — flip one in-bounds cell.
- `press(r, c)` — the player action: toggle the cross, increment `moves`, and
  check for a win.
- `lightsOn()` — count lit cells.
- `checkWin()` — win when `lightsOn() === 0`.

Because these are separate from rendering and all state is global, the Playwright
suite can set up an exact board, call `press`, and assert the outcome with no
timing dependence at all — Lights Out has no animation-driven logic.

## Controls

| Action              | Input                                   |
|---------------------|-----------------------------------------|
| Toggle a light      | Click / tap a tile                      |
| New board (same lvl)| **N**                                   |
| Reset current board | **R**                                   |
| Start / next level  | Click, any key, or the on-screen button |

## Assumptions

- **Novel game** — the repo (and its open pull requests) already cover Snake,
  2048, Tetris, Breakout, Asteroids, Pong, Space Invaders, Frogger, Minesweeper,
  Pac-Man, Missile Command, Lunar Lander, Tron, Doodle Jump, Centipede, Sokoban,
  Match-3, Bomberman, Bubble Shooter, Whack-a-Mole, Dino Run and Flappy Bird.
  Lights Out is a distinct grid-logic puzzle not present in any of them.
- Fixed 5×5 board and 400×400 canvas (no responsive resize) keeps click→cell
  mapping deterministic, matching the fixed-size approach of the other games.
- `best` tracks *fewest moves to clear a board* (lower is better) and shows `—`
  until the first solve, rather than a score that climbs.
- The generator regenerates if a random scramble happens to cancel out to an
  all-off (already-solved) board, so the player always starts with lights lit.
- The orphan `GeoDash/` HTML file in the repo root is not listed in the README,
  has no tests and is not marked *In Progress*, so it is left untouched.
